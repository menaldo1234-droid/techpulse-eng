# YouTube 영상 제작 가이드

> StackPulse 스터디 영상 제작을 위한 내부 참고 문서

## 전체 워크플로우

```
블로그 글 작성 → 스크립트 변환 → 촬영 → 편집 → 업로드 → StackPulse 연동
```

하나의 스터디 세션 = **YouTube 영상 1개 + StackPulse 블로그 글 1개**

## 1. 스크립트/대본 템플릿

### 기본 구조 (총 15-20분)

#### 인트로 (30초)
```
[얼굴캠]
"안녕하세요, 로봇 엔지니어 Henry입니다.
오늘은 [주제]를 공부해보겠습니다.
[한 줄 훅: 왜 이게 중요한지 / 어떤 삽질을 했는지]"
```

**훅 예시:**
- "이거 연결만 하면 된다더니, 한 달 걸렸습니다."
- "3만원짜리 센서가 이렇게 까다로울 줄 몰랐습니다."
- "공식 문서에는 3줄이면 된다는데, 현실은 달랐습니다."

#### 문제 정의 (1-2분)
```
[화면녹화 or 실물 보여주기]
- 오늘 풀려는 문제가 뭔지
- 왜 이걸 공부하게 됐는지 (프로젝트 맥락)
- 기대하는 결과물
```

#### 실습/삽질 과정 (10-15분)
```
[화면녹화 + PIP 얼굴캠]
- 실시간으로 코드 작성 / 하드웨어 조립
- 에러가 나면 그대로 보여주기 (편집으로 자르지 말 것)
- 생각 과정을 말로 설명하기
  "이렇게 하면 될 것 같은데..."
  "아, 이건 아니네. 에러 메시지를 보면..."
  "구글링 해보니까..."
```

**핵심 원칙: 실패 과정을 숨기지 않는다**
- 삽질 과정이 콘텐츠의 핵심 가치
- "30분 삽질했는데 안 됐습니다" → 타임랩스로 압축

#### 해결/정리 (3-5분)
```
[화면녹화 or 슬라이드]
- 최종 동작하는 결과 시연
- 핵심 포인트 3가지 이내로 정리
- 내가 헷갈렸던 부분과 해결법
- (선택) 표나 다이어그램으로 요약
```

#### 아웃트로 (30초)
```
[얼굴캠]
"오늘 내용 전체 정리는 블로그에 올려뒀습니다.
링크는 설명란에 있습니다.
다음에는 [다음 주제]를 공부할 예정입니다.
도움이 됐다면 구독 부탁드립니다."
```

### 스크립트 예시: ZED X Mini + Jetson

```
[인트로 - 얼굴캠]
"ZED X Mini 스테레오 카메라, Jetson에 연결만 하면 된다고 합니다.
저는 한 달 걸렸습니다. 오늘 그 이야기를 합니다."

[문제 정의 - 실물 카메라 보여주기]
"이 카메라는 GMSL2라는 프로토콜을 씁니다. USB가 아닙니다.
그래서 보드 선택, JetPack 버전, SDK 버전이 전부 맞아야 합니다."

[실습 - 화면녹화]
"먼저 공식 문서대로 해봤습니다... (설치 과정)"
"zed-explorer 실행했는데... 아무것도 안 뜹니다."
"에러도 없습니다. 그냥 카메라가 없다고 나옵니다."
(여러 시도 과정 보여주기)

[해결 - 화면녹화 + 정리표]
"결론적으로 이 조합이 됩니다: (표 보여주기)"
"핵심은 22-pin CSI 네이티브 보드를 써야 한다는 겁니다."

[아웃트로 - 얼굴캠]
"전체 트러블슈팅 과정은 StackPulse 블로그에 정리했습니다.
다음에는 ZED SDK로 depth map 추출하는 걸 해보겠습니다."
```

## 2. 장면 구성/스토리보드

### 카메라 앵글

| 앵글 | 용도 | 장비 |
|------|------|------|
| **얼굴캠** | 인트로, 아웃트로, 설명 | 웹캠 or 스마트폰 |
| **탑다운** | 보드 조립, 배선, 납땜 | 스마트폰 + 거치대 |
| **화면녹화** | 코드, 터미널, 브라우저 | OBS Studio |
| **클로즈업** | 커넥터, 핀 배열, LED 상태 | 스마트폰 매크로 |

### 화면 레이아웃

```
┌─────────────────────────────────┐
│                          ┌─────┐│
│     화면녹화 (코드/터미널)  │얼굴 ││
│                          │ 캠  ││
│                          └─────┘│
│  [하단] 현재 단계 텍스트 오버레이  │
└─────────────────────────────────┘
```

- **PIP (Picture-in-Picture)**: 화면녹화 시 우측 하단에 얼굴캠 작게 배치
- **풀스크린 얼굴캠**: 인트로/아웃트로, 중요한 설명 구간
- **풀스크린 화면녹화**: 코드 집중 구간, 터미널 출력 확인
- **분할 화면**: 코드(좌) + 결과(우) 비교

### 로봇공학 특화 장면

- **보드 전체 샷**: 어떤 보드를 쓰는지 한눈에 보여주기
- **핀 배열 클로즈업**: 연결하는 핀 번호가 보이도록
- **배선도 오버레이**: 실물 위에 화살표/라벨 그려넣기 (편집 시)
- **터미널 출력 확대**: `dmesg`, `lsusb` 등의 결과를 크게
- **Before/After**: 안 되는 상태 → 되는 상태 비교

### 장면 전환 흐름도

```
인트로(얼굴캠)
  ↓
문제 설명(실물 + 슬라이드)
  ↓
실습 시작(화면녹화 + PIP)
  ↓
삽질/에러(화면녹화, 에러 하이라이트)
  ↓
해결(화면녹화 + 정리표)
  ↓
결과 시연(화면녹화 or 실물)
  ↓
아웃트로(얼굴캠)
```

## 3. 촬영/편집 팁

### 녹화 도구

| 도구 | 용도 | 비고 |
|------|------|------|
| **OBS Studio** (무료) | 화면녹화 + PIP | 씬 전환 설정해두면 편함 |
| **웹캠** | 얼굴캠 | Logitech C920 이상 추천 |
| **마이크** | 음성 | USB 콘덴서 마이크 (FIFINE, Blue Yeti 등) |
| **스마트폰** | 탑다운, 클로즈업 | 거치대 필수 |
| **조명** | 얼굴 + 작업대 | 링라이트 or LED 패널 |

### OBS 기본 설정

```
녹화 설정:
- 해상도: 1920x1080
- FPS: 30
- 인코더: x264 or NVENC
- 비트레이트: 10,000-15,000 kbps
- 포맷: MKV (녹화 중 크래시 방지, 나중에 MP4로 리먹스)

씬 구성:
1. [얼굴캠 풀] - 인트로/아웃트로용
2. [화면 + PIP] - 코딩/터미널 작업용
3. [화면 풀] - 코드 집중 구간용
4. [탑다운] - 하드웨어 작업용 (캡처카드 or 스마트폰 연결)
```

### 로봇 엔지니어 촬영 팁

**작업대 세팅:**
- 배경 정리: 관련 없는 물건 치우기, 작업 중인 보드만 보이게
- 조명: 보드 위에 그림자 안 지게 LED 패널 45도 각도
- 카메라 높이: 탑다운은 보드에서 30-40cm 위

**터미널/코드 녹화:**
- 폰트 크기: 최소 16pt (시청자가 읽을 수 있게)
- 터미널 테마: 어두운 배경 + 밝은 글자 (Dracula, One Dark 등)
- 창 크기: 전체 화면의 80% 정도, 여백 남기기
- 중요한 명령어 입력 전 잠깐 멈추기 (시청자가 따라올 수 있게)

**하드웨어 촬영:**
- 흔들림 방지: 삼각대/거치대 필수
- 핀 번호가 보이도록 충분히 가까이
- 연결 전/후 각각 촬영
- LED 상태 변화도 촬영 (동작 확인용)

### 편집 워크플로우

**추천 편집 도구:**
- **DaVinci Resolve** (무료) - 전문적, 색보정 강력
- **CapCut** (무료) - 빠른 편집, 자막 자동생성
- **Premiere Pro** (유료) - 업계 표준

**편집 순서:**
1. 러프 컷: 불필요한 긴 침묵, 반복 삭제
2. 구조 정리: 인트로→실습→정리 순서 맞추기
3. 텍스트 오버레이: 핵심 키워드, 명령어, 에러 메시지 강조
4. 자막: 최소한 핵심 부분에라도 자막 추가
5. BGM: 잔잔한 로파이 or 없음 (코드 소리 중요)
6. 인트로/아웃트로 그래픽: 채널 로고, 구독 버튼
7. 최종 확인: 소리 레벨, 화면 전환, 오탈자

**시간 절약 팁:**
- 블로그 글을 먼저 쓰고, 그걸 스크립트 뼈대로 활용
- 긴 삽질 과정은 타임랩스 (4-8배속)
- 반복되는 인트로/아웃트로는 템플릿으로 만들어두기
- 자막은 CapCut 자동생성 → 수정이 가장 빠름

## 4. SEO/업로드 가이드

### 제목 공식

```
[핵심 키워드]: [결과 or 감정적 훅]
```

**좋은 예:**
- "ZED X Mini + Jetson: 30일간의 삽질 끝에 성공"
- "STM32 RTOS 입문: 처음부터 LED 깜빡이기까지"
- "ROS2 + Jetson: 로봇 자율주행 첫 걸음"
- "PCB 설계 첫 도전: KiCad로 센서 보드 만들기"

**피해야 할 예:**
- "공부 영상 #3" (검색 불가)
- "Jetson 세팅" (너무 막연)

### 설명란 템플릿

```
[주제]를 처음부터 공부하며 기록한 영상입니다.

📌 핵심 내용:
- [포인트 1]
- [포인트 2]
- [포인트 3]

⏱️ 타임스탬프:
00:00 인트로
00:30 문제 소개
02:00 실습 시작
08:00 에러 & 트러블슈팅
14:00 최종 결과
17:00 정리 & 다음 예고

📝 블로그 글 (전체 정리):
https://techblips.com/ko/study/[slug]

🔧 사용한 장비/소프트웨어:
- [보드명]
- [카메라/센서명]

#로봇공학 #임베디드 #[주제태그] #엔지니어스터디
```

### 태그 전략

**항상 포함할 태그:**
- 로봇공학, robotics, 임베디드, embedded
- 엔지니어, engineer, 스터디, study

**주제별 태그 (해당되는 것만):**
- Jetson, NVIDIA, CUDA, JetPack
- STM32, ARM, RTOS, FreeRTOS
- ROS, ROS2, 자율주행
- PCB, KiCad, 회로설계
- ZED, 스테레오카메라, depth sensing
- Python, C, C++

**태그 개수:** 10-15개 (YouTube 최대 500자)

### 썸네일 제작

**도구:** Canva (무료) or Figma

**썸네일 공식:**
```
┌──────────────────────────────────┐
│                                  │
│   [실물 하드웨어 사진]            │
│           +                      │
│   [큰 텍스트 2-3단어]            │
│           +                      │
│   [감정 표현 or 결과 아이콘]      │
│                                  │
└──────────────────────────────────┘
```

**규칙:**
- 해상도: 1280x720 (16:9)
- 텍스트: 최대 5단어, 멀리서도 읽히는 크기
- 색상: 밝은 배경 or StackPulse 시안(Cyber Teal #0d9488) 활용
- 하드웨어 사진: 실제 작업 중인 보드/센서
- 얼굴 사진 (선택): 표정이 있으면 클릭률 상승

**예시:**
- ZED 카메라 클로즈업 + "30일 삽질" 텍스트 + 좌절 이모지
- Jetson 보드 사진 + "첫 세팅 성공" + 체크마크

### 업로드 체크리스트

```
□ 영상 파일 (1080p, MP4)
□ 제목 (키워드 포함, 60자 이내)
□ 설명란 (타임스탬프 + 블로그 링크 + 태그)
□ 썸네일 (1280x720)
□ 태그 (10-15개)
□ 카테고리: 과학기술
□ 언어: 한국어
□ 자막 파일 업로드 (있으면)
□ 카드: 관련 이전 영상 링크
□ 엔드스크린: 구독 버튼 + 다음 영상
□ 공개 설정: 공개 (or 예약)
□ StackPulse 블로그 글 frontmatter에 youtube ID 추가
□ 블로그 글 본문에 영상 링크 삽입
```

## 빠른 참고: 첫 영상 최소 세팅

돈/장비가 부족하면 이것만으로 시작:

| 항목 | 최소 세팅 | 비용 |
|------|-----------|------|
| 화면녹화 | OBS Studio | 무료 |
| 얼굴캠 | 노트북 내장 웹캠 | 0원 |
| 마이크 | 이어폰 마이크 | 0원 |
| 편집 | CapCut or DaVinci Resolve | 무료 |
| 썸네일 | Canva | 무료 |

**완벽하지 않아도 됩니다. 시작하는 게 중요합니다.**
