# 고속 신호를 위한 다층 PCB 임피던스 제어

임피던스 불일치는 신호 반사를 일으켜 지터를 유발하고 데이터를 손상시키며, 소프트웨어 버그처럼 보이는 간헐적 장애를 만든다. 5Gbps 이상에서는 모든 트레이스가 전송선이며 모든 불연속점이 리스크다. 제어된 임피던스 라우팅, 의도적인 적층 설계, 제조 전 시뮬레이션은 필수다.

<!-- ![다이어그램: 임피던스 불일치 경계에서의 신호 반사](/images/pcb-impedance-reflection.png) -->

## 50MHz 이상에서 임피던스 제어가 필수인 이유

50MHz 이하에서 트레이스는 단순한 전선처럼 동작한다. 그 이상에서는 트레이스 폭, 유전체 높이, 기판 유전율에 의해 특성 임피던스가 결정되는 전송선이 된다. 2ns의 상승시간은 ~350MHz까지의 주파수 성분을 포함한다. 10GHz에서 1비트는 트레이스 위에서 약 1.3인치를 차지한다.

신호가 임피던스 불일치(50옴 트레이스가 45옴으로 좁아지는 경우)를 만나면 에너지가 역방향으로 반사되어 오버슈트, 언더슈트, 링잉이 발생한다. 임피던스 제어 없이는 트레이스 길이에 걸쳐 15~40%의 신호 열화를 예상해야 한다.

**흔한 오해:** 임피던스 매칭은 RF/아날로그 전용이 아니다. 상승시간이 1ns 미만이고 3인치 이상을 이동하는 디지털 신호라면 반드시 필요하다.

## 특성 임피던스 계산

**Z0 = 87 x log10(5.98 x H / W) / sqrt(Er)**

여기서 H = 유전체 두께, W = 트레이스 폭, Er = 비유전율.

| 신호 유형 | 목표 임피던스 | 허용 오차 |
|---|---|---|
| 단일 종단 디지털 | 50옴 | +/-5% |
| 차동 쌍 (PCIe, LVDS) | 100옴 차동 | +/-5% |
| 비디오/RF | 75옴 | +/-5% |

+/-10%를 초과하는 편차는 신호 무결성을 급격히 저하시킨다 -- 지터가 증가하고, 셋업-홀드 마진이 줄어들며, BER이 올라간다.

### 설계 형상 vs 제조 현실

0.5mm 유전체를 가진 4층 보드에서 50옴을 맞추려면 ~0.15mm 트레이스 폭이 필요하다. 0.1mm 유전체를 가진 8층 보드에서는 동일 임피던스에 ~0.08mm 트레이스가 필요하다 -- 47% 좁아지고, 공정 여유가 줄어들며, 비용이 20~30% 증가한다.

```python
import math

def microstrip_impedance(trace_width_mm, dielectric_height_mm, permittivity):
    ratio = (5.98 * dielectric_height_mm) / trace_width_mm
    return 87 * math.log10(ratio) / math.sqrt(permittivity)

def find_trace_width(target_z, dielectric_height_mm, permittivity):
    low, high = 0.01, 1.0
    while (high - low) > 0.001:
        mid = (low + high) / 2
        z0 = microstrip_impedance(mid, dielectric_height_mm, permittivity)
        if z0 < target_z - 0.1:
            high = mid
        else:
            low = mid
    return (low + high) / 2
```

## 적층 설계

### 3GHz 이상에 검증된 8층 적층 구조

| 레이어 | 유형 | 용도 |
|---|---|---|
| L1 | 신호 | 고속 차동 쌍 |
| L2 | 그라운드 | L1의 기준면 |
| L3 | 신호 | 보조 신호 |
| L4 | 전원 | VDD 분배 |
| L5 | 전원 | VSS 분배 |
| L6 | 신호 | 보조 신호 |
| L7 | 그라운드 | L6/L8의 기준면 |
| L8 | 신호 | 범용 |

### 비용 vs 성능

| 레이어 수 | 개당 비용 | 최대 주파수 | 용도 |
|---|---|---|---|
| 4 | $2-4 | ~500 MHz | 단순 디지털, 혼합 신호 |
| 6 | $4-8 | ~2 GHz | 중속 디지털 |
| 8 | $8-15 | 3+ GHz | 고속, 정밀 임피던스 |
| 10 | $15-25 | 3+ GHz | 고밀도, 다중 전원 도메인 |

**핵심 규칙:**
- 모든 신호 레이어에 인접한 그라운드 플레인을 배치하라 (0.1~0.2mm 간격)
- 적층을 중심선 기준으로 대칭 유지하여 휨을 방지하라
- 제조사에 확인하라: 최소 유전체 두께, 허용 오차(+/-10%), 비아 드릴링 정확도(+/-0.05mm)

## 트레이스 라우팅 규칙

- **리턴 비아:** 모든 신호 비아에서 0.5mm 이내에 배치 (중요 신호는 2~4개)
- **코너:** 45도 각도 사용, 90도는 절대 금지
- **차동 쌍 매칭:** 0.25mm (10mil) 이내
- **크로스토크 간격:** 그라운드 위 트레이스 높이의 최소 3배 (0.1mm 높이에 0.3mm; 0.5mm이면 결합을 ~60% 줄임)

## 종단 전략

| 유형 | 메커니즘 | 소비 전력 | 적합 용도 |
|---|---|---|---|
| 직렬 | 드라이버 측 25~50옴 저항 | 최소 | 점대점 연결 |
| 병렬 | 수신기 측 전원/그라운드 저항 | 3.3V/50옴 기준 ~66mW | 멀티드롭 버스 |
| AC | 커패시터 + 저항 (DC 차단) | ~5mW (13배 적음) | 전력 제한 설계 |

수신기에서 12mm 이내에 종단을 배치하라. 저항값을 트레이스 임피던스에 맞추라.

## 비아 전환

비아 패드는 트레이스 폭 + 측면당 0.1mm 클리어런스에 맞춰야 한다. 과대한 패드는 5~15옴의 불일치를 만든다. 5GHz 이상 신호에서는 미사용 스터브를 백드릴하라 (기생 커패시턴스 30~40% 감소, 보드 비용 15~20% 증가).

## 검증 워크플로

1. **시뮬레이션:** 목표 주파수에서 2D 단면 전자기 해석을 실행한다. 임피던스가 +/-10% 이내인지 확인한다.
2. **과도 해석:** 드라이버, 트레이스, 수신기를 모델링하여 아이 다이어그램을 생성한다. 정상적인 3.2GHz 차동 설계는 200mV 수직 마진과 200ps 수평 마진을 보인다.
3. **제조 보드 측정:** 네트워크 분석기로 S-파라미터를 추출한다. 시뮬레이션과 5~8% 수준의 일치를 기대한다.
4. **TDR 검사:** 시간 영역 반사법으로 비아 전환부의 숨겨진 임피던스 불연속을 발견한다.

## 문제 해결

| 증상 | 원인 추정 | 해결 방법 |
|---|---|---|
| 특정 클록 속도에서 간헐적 비트 오류 | 임피던스 불일치 또는 종단 불량 | TDR 측정, 종단값 확인 |
| 신호 진폭의 20% 초과 오버슈트/언더슈트 | 감쇠 부족 반사 | 드라이버 측에 22~47옴 직렬 저항 추가 |
| 온도에 따라 악화되는 타이밍 위반 | 유전 손실 또는 트레이스 저항 드리프트 | 트레이스 길이 단축, 저손실 소재 사용 |
| 인접 신호의 크로스토크 | 간격 부족 | 높이의 3배 간격 유지, 다른 레이어에 라우팅 |
